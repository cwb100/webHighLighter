# Web Highlighter

一个用于 Chrome / Edge 的网页划线扩展，支持在网页中高亮文本、持久化保存、恢复高亮，并通过独立 dashboard 管理所有标记。
![alt text](./assets/image1.png)

## 当前支持的功能

- 在任意网页中选中文本后，使用悬浮工具条快速高亮
- 支持 5 种预设颜色
- 支持在高亮操作菜单中使用自定义颜色
- 高亮结果会持久化保存到扩展侧的 IndexedDB
- 刷新页面后可恢复高亮
- 支持跨行、跨段落高亮，尽量不破坏原页面结构
- 优化了换行场景下的高亮显示，尽量减少行首碎片和结构感破坏
- 支持普通 DOM 文本与部分 Shadow DOM 文本的高亮
- 点击已高亮文本后可打开操作菜单
- 支持删除单条高亮
- 支持修改单条高亮颜色
- 页面右侧会显示与高亮颜色对应的导航标记
- 点击右侧导航标记，可快速跳转到对应高亮位置
- 鼠标悬浮在右侧导航标记上时，会显示对应高亮文本；长文本会自动省略
- 右侧位置过近的多个高亮会聚合为一组导航标记
![alt text](./assets/image2.png)
- 提供独立 dashboard 页面，集中查看所有已保存的网页和标记
- dashboard 支持点击单条标记，在新标签页打开对应网页
- dashboard 支持删除单条高亮和清空某个页面的全部高亮
- dashboard 支持多字段筛选：标题、网址、标记文本、颜色、日期
- dashboard 支持导出 JSON 备份文件，用户可通过浏览器下载能力选择保存位置
- dashboard 支持导入 JSON 备份文件，用于跨浏览器、跨电脑或跨平台迁移
- 导入备份时会与现有数据合并，并按高亮 ID 去重，即只会新增不同的部分
- 内容页与 dashboard 共用扩展侧存储，新建、删除、改色后 dashboard 可同步刷新

## 适配说明

- 当前版本优先面向普通文章页、资讯页、长文本页，以及部分动态站点
- 对于 Bilibili 这类单页应用页面，已增加路由变化和延迟渲染后的恢复机制
- 对于视频页、评论区这类动态渲染区域，已增加针对评论文本选择与恢复的兼容处理
- 高亮恢复依赖页面文本内容和页面结构在下次打开时仍然足够接近原始状态
- 如果页面内容变化过大、标题被平台替换、或站点进行了强重渲染，个别高亮仍可能无法精确恢复

## 存储方式

- 插件运行时主存储是扩展自己的 IndexedDB，数据库名为 `webHighlighterDB`
- IndexedDB 的物理路径由浏览器 Profile 管理，用户通常不应直接读写或移动该目录
- 用户需要自行选择保存位置时，应在 dashboard 中使用“导出数据”生成 JSON 备份文件
- 用户更换电脑、浏览器或平台时，可在新环境中使用“导入数据”合并导入备份文件
- dashboard 和内容页通过后台脚本访问同一份 IndexedDB 数据

## 项目结构

- `manifest.json`：扩展配置
- `background.js`：扩展后台，负责打开 dashboard 和代理存储请求
- `storage.js`：IndexedDB 存储层、旧数据迁移、备份导出与导入逻辑
- `content.js`：网页内高亮、恢复、动态页面监听与页面交互逻辑
- `dashboard.html` / `dashboard.js` / `dashboard.css`：独立管理页
- `icons/`：扩展图标资源，用于浏览器工具栏、扩展列表和商店包

## 安装方式

1. 打开 Chrome 或 Edge
2. 进入 `chrome://extensions` 或 `edge://extensions`
3. 开启“开发者模式”
4. 选择“加载已解压的扩展程序”
5. 选择本项目目录

## 使用方法

1. 在网页中选中一段文本
2. 在出现的悬浮工具条中选择颜色
3. 高亮会立即写入存储
4. 点击已高亮文本，可以删除或修改颜色
5. 页面右侧会出现对应颜色的导航标记，可悬浮查看标记内容，也可点击快速跳转到高亮处
6. 点击扩展图标，打开 dashboard 查看全部记录
7. 在 dashboard 中通过标题、网址、标记文本、颜色、日期进行筛选
8. 点击 dashboard 中的单条标记，可在新标签页打开对应网页
9. 点击 dashboard 顶部“导出数据”，保存 JSON 备份文件
10. 点击 dashboard 顶部“导入数据”，选择 JSON 备份文件并合并导入

## 已知限制

- 高亮恢复不是 DOM 级快照恢复，而是基于文本内容、偏移和上下文重新定位
- 当网站内容发生明显变化时，旧高亮可能无法恢复到原位置
- 极度动态的站点仍可能出现恢复延迟，通常会在后续重试中补上
- 评论区等复杂动态区域的兼容性仍可能因站点实现差异而不稳定
- 右侧导航标记在高亮特别密集的区域会做聚合显示，而不是逐条完全展开
- 浏览器扩展不能像桌面应用一样持续读写任意用户指定路径，跨平台迁移通过导出/导入 JSON 完成
